Exercise 607 - Janitor and CSS

This exercise is about developing your first Janitor based PHP project

The purpose of this exercise is to work with your personal Janitor project to gain experience with Janitor and CSS.

Tasks

  1. Understand the CSS folder

    Your CSS lives in theme/www/css. As default Janitor expects you to use Detector segments and this is reflected in the default file structure.

    Inside the theme/www/css folder you will find two folders, lib and fonts and some files like seg_desktop.css and seg_smartphone.css. The CSS files in this folder are merged segment-files, that has been generated by the system and should never be edited.

    The lib folder is where you write your CSS. In the lib folder you find a desktop and a smartphone folder. They are intended for desktop and smartphone CSS respectively. This is where you will place your CSS, preferably structured in multiple files.

    It is normal to start your project with an s-page.css, an s-header.css, an s-content.css and an s-footer.css containing the generalized CSS of your site. Deviations should be placed in either an s-pages.css (if there are only few deviations), or in individual files named by the module/page they cover.

    In the lib folder, you will find an s-fonts.css file. This contains the font specifications and this file is almost always shared between the different segments.

    In the lib folder, you will also find some files like seg_desktop_include.css and seg_smartphone_include.css. These files contain the include settings for their segment â€“ in this the desktop or smartphone segments respectively. The segment support can be expanded to cover all 12 Detector segments, but in most cases less will do, by grouping segments to fit your needs, as explained in exercise 405.

  2. Modify the desktop page CSS

    In theme/www/css/lib/desktop you will find an s-page.css file, containing the base desktop CSS of your site. Make some changes to the CSS and see what happens.

  3. Modify the smartphone page CSS

    In theme/www/css/lib/smartphone you will find an s-page.css file, containing the base smartphone CSS of your site. Make some changes to the CSS and see what happens.

  4. Customize your site

    Add some CSS to your new templates or rewrite the entire CSS for the site. Or something in-between. Mix-in some JavaScript if you feel like it.

  5. Remember

    Never use inline CSS in your templates. Ever. No.
