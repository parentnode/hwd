So lets talk a bit about structure and why its is so important, First of all most html documents typically consist of a "header" a "footer" and something in between, lets call the in between "content". Just look at source code of this page (right click -> source). Here you can see the div's within div's. right "above" the "content div" is an empty "header div", and "below" the content is an "empty" "footer div". they are all on the same "level" in the if you will. or they are siblings. They have the same parent div with the "id = page". you can see that there is a body tag that has a sibling called head. These to children body and head has a parent called html. there you have the whole family. Do you notice a system here? Everytime there is a child or children there is an indentation. and indentation happens is when you press the tab button in a document, sit on the left side of your keyboard left to the letter Q. it makes 4 spaces infront of the letter behind it like so:

indent example

note that the html tag is not indented. That is convention. That is because everyone has agreed to not indent everything to the right one extra time for no real reason. in that way all the divs in the html document has a better chance of staying inside the screen. So you could acually write html with making indentation at all it would just become very confusing. while we are at it remember that tags are always above each other. here is how to do it <div class="outer_div"> <div class="Im_a_div" > <div class= "long" > <p> Im a long text that goes farrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrrr note how all my text and my other tags inside are indented and I would like my programmer to se my end tag. <span>Im a span in a paragraph </span> </p> </div> <div class= "short" > <p>im a short text only one line</p> </div> </div> </div>

Why is the header before the content? The header is for al sort of information that should be seen first. Like if you where a blind person using this it would be nice there was a phone number, or email address. If you are blind you aplication the read to you what is happening at the screen will will most likely start from the top. html is loaded from top to bottom so in the case you have a really slow computer or/and internet access you will also want to load the most important things on top like contact detail.
