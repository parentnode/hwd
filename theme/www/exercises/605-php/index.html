Exercise 605 - Janitor and Detector

This exercise is about developing your first Janitor based PHP project

The purpose of this exercise is to work with your personal Janitor project to gain experience with Janitor and Detector segments.

Tasks

  1. Understand the Detector segments

    Janitor natively supports Detector segments. While Janitor can also be used entirely without segments it is recommended to use the segments to provide optimized UI's for each segment, rather than one size fits all.

    Janitor will automatically detect the segment of the visiting browser, by making a request to the Detector API, when the $page->segment method is called. As default this happens in the templates/www.header.php template, where the segment of the current user's browser is used to include the correct JavaScript and CSS. The detection process only happens once per site visit and the detected segment is stored in the user session for the duration of the visit.

    Janitor accepts a segment=#segment-value# GET parameter on any page, which will force the browser to run under the specified segment. Fx. /?segment=smartphone will force the site to show the smartphone UI.

    In addition Janitor accepts a dev=1 GET parameter on any page, which will force the site to load the development code, rather than the generated production code. This parameter simply indicates whether to include the system generated JavaScript and CSS bundles or the individual source files. Use dev=1 when developing for better debugging. The opposite of dev=1 is dev=0. You can observe the difference using the Network pane of your browsers developer tools.

    You will learn more about how to structure your JavaScript and CSS and how to create production bundles in the following exercises.

  2. Defining the segment support

    The exact segment support can be individualized for each Janitor project. You can choose to use all the original 12 segments individually for an extensive browser support â€“ but you can also group relevant segments to fit the exact browser support needs of your project and minimize the development effort.

    We differentiate between the 12 original segments from Detector (available segments) and the segments or group segments you choose to support in your project (supported segments). Although Detector will always return the original Detector segment for any given browser, you can customize which segment values $page->segment can return in your project.

    In theme/config/segments.php you will find the settings for your current project in an associative Array. You will find that the keys represent the available segments, and the values specify the supported segments. In other words, it defines which segment will be applied if a desktop_light browser visits your page - in this case unsupported. Likewise, smartphone stays smartphone and tablet joins the desktop segment.
