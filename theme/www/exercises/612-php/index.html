Exercise 612 - Testing smartphones

This exercise is about how to test your project on a smartphone.

The purpose of this exercise is to work with your personal Janitor project to gain experience with testing on smartphones.

Tasks

  1. Install Squid proxy server

    We use a proxy server to enable testing your code on your smartphone devices.

    Find and follow the guide for setting up Squid on parentnode.dk/blog.

    The following tasks are also described in the post, and are listed here to add further explanation.

  2. Update your hosts file

    Open your hosts file “/etc/hosts” in VS Code and find your test domain. Add a .proxy variant of your test domain in a new line, using the local IP of your computer. You can find your local IP by running ifconfig in Bash. It may prompt with an installation suggestion “net-tools”. If so just install it with the command: sudo apt install net-tools. Then run the “ifconfig” command once again. The command will generate a lot of output and the lines you are looking for is very likely starting with inet 192.168.x.x or inet 10.0.x.x. Assign this IP with your test domain with a .proxy variant and save the changes in the file.

  3. Update your local project Apache configuration

    Add a ServerAlias to your project Apache configuration: Open project-name/apache/httpd-vhosts.conf file in VS Code. Add a .proxy variant of your domain. You can add it right under ServerName at the upper part of the file.

  4. Restart Apache

    Restart Apache to enable the new settings with the Bash command: apache restart

  5. Configure proxy on your smartphone

    Follow the last element of the guide, to configure the proxy server on your smartphone.

    Visit the .proxy variant of your domain on your smartphone and you should be able to see the project running on your local computer.
