Exercise 408 - Janitor CSS/JavaScript production bundles

This exercise is about developing your first Janitor based PHP project

The purpose of this exercise is to work with your personal Janitor project to gain experience with Janitor.

Tasks

  1. Understand the merging / bundling process

    Your JavaScript and CSS has been structured for an optimized workflow with the intention to merge the individual JavaScript and CSS files into just two files. One for JavaScript and one for CSS. Per segment. One reason for this is to minimize the requests made by your site.

    The js/lib/seg_#segment#_include.js and css/lib/seg_#segment#_include.css files contain all the information to build each segment bundle, by merging all the files listed in each include into one file. After building your production bundles, you will see the result in js/seg_#segment#.js and css/seg_#segment#.css. One JavaScript file and one CSS file for each segment you support.

    The actual building process is handled by scripts in two separate submodules, js-merger and css-merger.

  2. Build bundles

    To build the production bundles, you have to request the building script through your browser. Simply visit #your-domain#.local/js/lib/build to build the JavaScript bundles and #your-domain#.local/css/lib/build to build the CSS bundles.

    When you visit these pages, they will output information about the building process. You can click on each file in the view to see more details.

    If there are any errors in the output, then fix the errors and build again. The building process is not complete until the scripts have been run without errors.

    When the process is done, check the result in your Git application and commit the updated files.

  3. Janitor backend JavaScript and CSS

    The backend interface of Janitor is also using Detector and Manipulator, which also needs to be built after install and each time you upgrade Janitor. You also need to build again if you add custom code to either theme/www/janitor/js or theme/www/janitor/css.

    To build the Janitor backend bundles visit #your-domain#.local/janitor/js/lib/build and #your-domain#.local/janitor/css/lib/build.
