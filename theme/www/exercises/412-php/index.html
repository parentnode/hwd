Exercise 412 - Testing smartphones

This exercise is about how to test your project on a smartphone.

The purpose of this exercise is to work with your personal Janitor project to gain experience with testing on smartphones.

Tasks

  1. Install Squid proxy server

    We use a proxy server to enable testing your code on your smartphone devices.

    Find and follow the guide for setting up Squid on parentnode.dk/blog.

    The following tasks are also described in the post, and are just listed here to ensure you get all the way through.

  2. Update your hosts file

    Open your hosts file and add a .proxy variant of your test domain, using the local IP of your computer as described in the guide. You can find your local IP by running ifconfig in Bash. It is very likely starting with inet 192.168.x.x or inet 10.0.x.x.

  3. Update your local project Apache configuration

    Add a ServerAlias to your project Apache configuration (project-name/apache/httpd-vhosts.conf) using a .proxy variant of your domain.

  4. Restart Apache

    Restart Apache to enable the new settings.

  5. Configure proxy on your smartphone

    Follow the guide to configure the proxy server on your smartphone.

    Visit the .proxy variant of your domain on your smartphone and you should be able to see the project running on your local computer.
