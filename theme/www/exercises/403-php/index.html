Exercise 403 - PHP

This exercise is about developing your first Janitor based PHP project

The purpose of this exercise is to work with your personal Janitor project to gain experience with frameworks and PHP.

Tasks

  1. Add controller endpoints

    Every part (separated by /) of the URL you request is either a controller or a REST parameter for the controller. Fx. if your controller is called test.php, then a request to /test/help, will pass the parameter help to the test.php controller via the $actions array.

    Modify the controller you added in exercise 403 and add controller logic so that the controller is able to respond to two parameters. If you send test1 it should show a different template than if you send test2. It should still maintain the default output if no parameters are sent. Create additional templates as needed.

    You can use the posts.php controller as inspiration. This controller is able to act differently depending on which parameters it receives. If it has just one parameter, then it will show the pages/post.php template. If it has two parameters and the first one is tag, then it will show the pages/posts_tag.php template. Otherwise it will just show the pages/posts.php template, which is the default view of this controller.

  2. Visit your new page

    Visit the page you have created and test how it works when you send the new parameters. If your controller is named test.php, then visit /test, /test/test1 and /test/test2 on your local project domain.
